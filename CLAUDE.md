# プロジェクトルール

## Git運用

- このリポジトリは他の人と同時に作業している共同プロジェクト。
- **`git push` は絶対に行わない。** ブランチの作成・コミットまでに留める。
- 本番反映が必要な場合は、ユーザーが手動でブランチをpushし、GitHub上でPull Requestを作成してマージする（Claudeはpushしない）。
- 明示的にユーザーから「pushして」と指示された場合のみ、内容を確認した上でpushしてよい。
- 作業ブランチ（例: `komatsu`）で新しく作業を始める前は、必ず `git fetch origin` した上で `git log <作業ブランチ>..origin/main --oneline` などで `main` が先行して更新されていないか確認する。更新があれば、作業を始める前に `git merge origin/main` を作業ブランチに取り込んでから進める（後から大きなコンフリクトになるのを防ぐため）。

## 変更履歴の記録

- ブランチに変更をコミットした後（pushやpullで他ブランチ・mainに取り込むタイミング）は、`model_notes.md` の一番下に変更内容をタイムスタンプ付きで時系列に追記する。
- 形式は以下を基本とする：

  ```
  ### YYYY-MM-DD HH:MM
  - 変更内容の要約（コミットメッセージ相当）
  ```

- 既存の記載は書き換えず、常に最後に追記していく（時系列ログとして残す）。
