## 1. これは何か

塾の生徒が、スマホ・タブレットから勉強の質問を送ると、AI（Google Gemini）が家庭教師のように答えてくれるWebチャット。
質問ボックスの代わりとして、生徒がいつでも質問できることを目的にしている。

- 想定ユーザー: 中学生（中1〜中3）。各自が自分の端末で使う。
- 公開URLのWebサイト（Vercelでホスティング）。

---

## 2. 技術構成・ファイル

| 部分 | 技術 | 役割 |
| --- | --- | --- |
| フロント（画面） | HTML/CSS/JS 1ファイル | `public/index.html`。チャット画面、名前/学年/クラス入力、画像送信、KaTeXで数式表示 |
| バックエンド（中継） | Vercel Serverless Function | `api/chat.js`。Gemini APIへ質問を中継し、回答を返す。ログも送る |
| ログ保存 | Google Apps Script + スプレッドシート | `log.gs`。質問・回答などを1行ずつ記録 |

使用モデル: **`gemini-2.5-flash`**（Gemini API）。

APIキーなどの秘密情報は **Vercelの環境変数** に入れる（コードに直書きしない）。
- `GEMINI_API_KEY` … Gemini APIのキー
- `LOG_WEBHOOK_URL` … Apps ScriptのウェブアプリURL（`/exec` で終わるもの）

## 3. デプロイの流れ（重要）

**反映ルートが2つに分かれている点に注意。**

### A. サイト本体（画面＋AIの動作）
`public/index.html` と `api/chat.js` の変更は、`main` ブランチにpushするとVercelが自動でデプロイする。

```
cd juku-ai-chat
git add .
git commit -m "変更内容"
git push origin main
```

push後、1〜2分でVercelに反映。動作確認するときはブラウザのタブを開き直す（後述のsessionStorageの都合）。

### B. ログ用のApps Script（`log.gs`）
これは **Vercelとは別物**。GitHubにpushしてもログ側は更新されない。
`log.gs` を変えたら、Google Apps Scriptのエディタに貼り直して保存し、「デプロイを管理 → 編集 → 新バージョン」で再デプロイする必要がある。

### ハマりどころ
- 本番は `main` ブランチ。
- GitHubサイト上で直接編集することもあるので、ローカルで作業する前に `git pull` してから始めると安全（pushが弾かれる原因になる）。
- Apps ScriptのURLは末尾が `/exec` のものを使う（`/dev` ではない）。

## 4. いまある機能

### 生徒の画面
- **名前・学年・クラスの入力**: 最初にモーダルで入力。学年は中1〜中3。クラスは選択式。
  - 中3 → `S（御三家志望）` / `A` / `B` / `個別`
  - 中1・中2 → `A` / `B` / `個別`（Sは中3だけ表示）
  - 右上の「名前変更」でいつでも変更可。
- **入力の保持は sessionStorage**: タブを開いている間とリロードでは名前を覚えているが、**タブを閉じると次回は必ず聞き直す**。共用端末でも他人の名前が固定されないようにするため。
- **テキスト質問**＋**画像送信**（カメラ撮影・アルバムの写真どちらも選べる）。
- **数式表示**: AIの回答内の数式はKaTeXできれいに表示（`$...$`、`$$...$$`）。
- **「🍀 もっとやさしく説明して」ボタン**: 直前の説明をもっとやさしく短く言い直してもらう。
- **混雑時メッセージ**: レート制限（429）に当たると「1分くらい待ってね」と表示。

### バックエンド（`api/chat.js`）
- フロントから `studentName / studentGrade / studentClass / message / 画像 / 直近履歴` を受け取る。
- AIへの指示（システムプロンプト）＋クラス別の指導方針を組み立ててGeminiに投げる。
- 直近の会話履歴を最大 **6往復**（`MAX_HISTORY = 6`）まで一緒に送って文脈を保つ。
- 429（混雑）/ その他エラーを生徒向けのやさしい文言にして返す。
- 回答が返ったらログをApps Scriptに送る。

## 5. AIへの指示（プロンプト）の方針

実体は `api/chat.js` の `SYSTEM_PROMPT`（全生徒共通）と、`buildLevelInstruction()`（学年・クラス別に追加）。
中身を変えるときはこの2か所を編集する。

### 共通方針（SYSTEM_PROMPT）の要点
- **答えを丸ごと教えない**。ヒントと「次の一歩」で自分で解かせる。最後の答え合わせは生徒が出してから。
- **返信は短く**（3〜5行・150字程度）。一度に1ステップだけ。次は生徒が反応してから。
- **やさしい言葉**で。専門用語は避け、中学生の言葉で。
- **質問は控えめ**。いきなり質問返しせず、まず小さなヒント→最後に「はい/いいえ」で答えられる簡単な質問を1つだけ。
- **生徒に式を打たせない**。式はAIが書いて「合ってる？」と確認、または番号で選ばせる。書いた式を見たいときは写真を促す。
- **途中式の写真を要求**: 計算問題は、答え合わせの前に必ず一度「ノートの写真を送ってね」と求める（自分で解いた確認のため）。用語の質問では不要。
- **AIの限界を正直に**: 不確かなときは断定せず、難しければ「先生に聞いてね」と促す。
- **勉強以外・不適切な内容には応じない**。

### クラス別の指導方針（buildLevelInstruction）
| クラス | 方針 |
| --- | --- |
| S（中3・御三家志望） | ヒント最小限。自分で考えさせる時間を長く。本質や発展的な見方にも触れる。答えは丸ごと教えない |
| A（上位） | 基礎はできている前提。標準〜応用。ヒントやや少なめ |
| B（基礎） | 一歩を小さく、言葉を特にやさしく、例を多めに。よくほめる |
| 個別 | その子のペース最優先。つまずきを一つずつ丁寧に。ゆっくり |

加えて、学年（中1〜中3）が習う範囲を超えた難しすぎる解法は避けるよう指示している。

## 6. 主な設定値（変えたいとき用）

`api/chat.js`:
- モデル: `gemini-2.5-flash`
- `temperature`: `0.3`（低いほど回答が安定・固め）
- `maxOutputTokens`: `1024`（※短さは基本プロンプト側で制御。ここを下げると文が途中で切れることがある）
- `MAX_HISTORY`: `6`（送る直近の会話往復数）

`public/index.html`:
- `MAX_IMAGE_PX`: `1600`（送信前に画像をこのサイズに縮小）

## 7. データとプライバシー

- **ログに残るもの**（スプレッドシート）: 日時 / 生徒名 / 質問 / 画像あり・なし / AI回答 / トークン数 / 学年 / クラス。
  - ※スプレッドシートのヘッダーは I列「学年」・J列「クラス」を追加する必要あり。
- **画像そのものは保存していない**: フロントは送信後にメモリから消える。ログには「あり/なし」しか残さない。Vercelにもファイル保存しない。
- **Google（Gemini）側**: 質問・画像はAIが答えるためにGoogleへ送られる。
  - **無料プランでは、送った内容がGoogleの製品改善（AI学習）に使われる**。有料プランでは使われない。
  - → 生徒の個人情報やテスト答案など、写り込んでまずいものは送らせない案内をしておくと安心。

---

## 8. 料金・レート制限

- 現在は **無料プラン**。`gemini-2.5-flash` は無料枠では入力・出力ともに無料。
- **上限を超えても自動課金はされない**。無料プランには回数制限（例: 1分あたりの上限）があり、超えるとリクエストがエラー（429）で一時的にはじかれるだけ。料金は発生しない。
- **課金が始まるのは、自分でクレジットカードを登録して有料プランにアップグレードしたときだけ**。有料にすると上限が大きく上がる（従量課金: 入力 $0.30 / 出力 $2.50 per 1Mトークン）。
- レート上限はAPIキー（サイト全体）単位の合計。同時利用が増えて足りなくなったら有料化を検討。
