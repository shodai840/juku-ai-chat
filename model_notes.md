## 1. これは何か

塾の生徒が、スマホ・タブレットから勉強の質問を送ると、AI（Google Gemini）が家庭教師のように答えてくれるWebチャット。
質問ボックスの代わりとして、生徒がいつでも質問できることを目的にしている。

- 想定ユーザー: 中学生（中1〜中3）が中心。高校生も利用可（学年は分けず「高校生」）。各自が自分の端末で使う。
- 公開URLのWebサイト（Vercelでホスティング）。

---

## 2. 技術構成・ファイル

| 部分 | 技術 | 役割 |
| --- | --- | --- |
| フロント（画面） | HTML/CSS/JS 1ファイル | `public/index.html`。チャット画面、名前/学年/クラス入力、画像送信、KaTeXで数式表示 |
| バックエンド（中継） | Vercel Serverless Function | `api/chat.js`。Gemini APIへ質問を中継し、回答を返す。ログも送る |
| ログ保存 | Google Apps Script + スプレッドシート | `log.gs`。質問・回答などを1行ずつ記録 |

使用モデル: **`gemini-2.5-flash`**（Gemini API）。

APIキーなどの秘密情報は **Vercelの環境変数** に入れる（コードに直書きしない）。
- `GEMINI_API_KEY` … Gemini APIのキー
- `LOG_WEBHOOK_URL` … Apps ScriptのウェブアプリURL（`/exec` で終わるもの）

## 3. いまある機能

### 生徒の画面
- **名前・学年・クラスの入力**: 最初にモーダルで入力。学年は中1 / 中2 / 中3 / 高校生。
  - 中3 → クラス `S（御三家志望）` / `A` / `B` / `個別`
  - 中1・中2 → クラス `A` / `B` / `個別`（Sは中3だけ表示）
  - 高校生 → 学年は細かく分けず「高校生」の1択。クラスなし（クラス欄は自動で隠れ、選択不要）。先取り学習の子がいるため学年で範囲を縛らず、AIは質問内容からレベルを判断する。
  - 右上の「名前変更」でいつでも変更可。
- **入力の保持は sessionStorage**: タブを開いている間とリロードでは名前を覚えているが、**タブを閉じると次回は必ず聞き直す**。共用端末でも他人の名前が固定されないようにするため。
- **テキスト質問**＋**画像送信**（カメラ撮影・アルバムの写真どちらも選べる）。
- **数式表示**: AIの回答内の数式はKaTeXできれいに表示（`$...$`、`$$...$$`）。
- **「🍀 もっとやさしく説明して」ボタン**: 直前の説明をもっとやさしく短く言い直してもらう。
- **混雑時メッセージ**: レート制限（429）に当たると「1分くらい待ってね」と表示。

### バックエンド（`api/chat.js`）
- フロントから `studentName / studentGrade / studentClass / message / 画像 / 直近履歴` を受け取る。
- AIへの指示（システムプロンプト）＋クラス別の指導方針を組み立ててGeminiに投げる。
- 直近の会話履歴を最大 **6往復**（`MAX_HISTORY = 6`）まで一緒に送って文脈を保つ。
- 429（混雑）/ その他エラーを生徒向けのやさしい文言にして返す。
- 回答が返ったらログをApps Scriptに送る（日時・生徒名・質問・画像あり/なし・AI回答・トークン数・学年・クラス）。画像そのものは保存しない。

## 4. AIへの指示（プロンプト）の方針

実体は `api/chat.js` の `SYSTEM_PROMPT`（全生徒共通）と、`buildLevelInstruction()`（学年・クラス別に追加）。
中身を変えるときはこの2か所を編集する。

### 共通方針（SYSTEM_PROMPT）の要点
- **答えを丸ごと教えない**。ヒントと「次の一歩」で自分で解かせる。最後の答え合わせは生徒が出してから。
- **返信は短く**（3〜5行・150字程度）。一度に1ステップだけ。次は生徒が反応してから。
- **やさしい言葉**で。専門用語は避け、中学生の言葉で。
- **質問は控えめ**。いきなり質問返しせず、まず小さなヒント→最後に「はい/いいえ」で答えられる簡単な質問を1つだけ。
- **生徒に式を打たせない**。式はAIが書いて「合ってる？」と確認、または番号で選ばせる。書いた式を見たいときは写真を促す。
- **途中式の写真を要求**: 計算問題は、答え合わせの前に必ず一度「ノートの写真を送ってね」と求める（自分で解いた確認のため）。用語の質問では不要。
- **AIの限界を正直に**: 不確かなときは断定せず、難しければ「先生に聞いてね」と促す。
- **勉強以外・不適切な内容には応じない**。

### クラス別の指導方針（buildLevelInstruction）
| クラス | 方針 |
| --- | --- |
| S（中3・御三家志望） | ヒント最小限。自分で考えさせる時間を長く。本質や発展的な見方にも触れる。答えは丸ごと教えない |
| A（上位） | 基礎はできている前提。標準〜応用。ヒントやや少なめ |
| B（基礎） | 一歩を小さく、言葉を特にやさしく、例を多めに。よくほめる |
| 個別 | その子のペース最優先。つまずきを一つずつ丁寧に。ゆっくり |
| 高校生（クラスなし） | 学年で範囲を縛らず質問内容でレベル判断。発展的な内容にも対応。答えは丸ごと教えない |

加えて、中学生では学年（中1〜中3）が習う範囲を超えた難しすぎる解法は避けるよう指示している。

## 5. 今後の検討メモ（未実装）

- **教科ごとのルール・教え方**: 数学は途中式の写真必須、社会は写真不要、など教科別に対応を変える案。判別方法は「生徒が教科ボタンで選ぶ（確実）」か「AIが自動判別（手軽）」の2案。`buildSubjectInstruction(教科)` を足してプロンプトに合体させるイメージ。→ まずは現状のまま生徒に使ってもらい、反応を見てから判断する。
- **生徒名簿のオートコンプリート**: 名前の表記ゆれやクラス選び間違いを防ぐ案。ただし「全生徒一覧を公開URLで見せる」のはプライバシー上NG。名前を打つと候補が出て確定＋クラスは登録済みを自動入力、という折衷案が候補。名簿はGoogleスプレッドシートで管理してサイトが読み込む構成を想定。今は自由入力のまま運用中。
- 類題の出題、回答末尾の「今日わかったこと」まとめ なども会話で挙がったアイデア（未着手）。
